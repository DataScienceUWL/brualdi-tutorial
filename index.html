Brualdi’s conjecture — a from-scratch tutorial

A from-scratch tutorial on Brualdi’s interchange-graph conjecture: starting from “what is a flip graph?” and building, with worked examples, up to a recent candidate resolution — what is proved, how, and what remains to be verified. It assumes comfort with undergraduate linear algebra and a little probability, but no combinatorics: every term is defined on first use and small cases are worked by hand. Short, self-contained parts.

Part 1 — Graphs, in 3 minutes

A graph is just a set of vertices (dots) and a set of edges (lines joining pairs of dots). That’s it. No directions, no repeated edges for us — a “simple graph.”

Two vertices joined by an edge are adjacent (or neighbours). The degree of a vertex is how many neighbours it has.

Tiny example — 4 vertices in a square:

A --- B
|     |
C --- D

Vertices {A,B,C,D}; edges {AB, AC, BD, CD}. Every vertex has degree 2. This square graph has a name we’ll reuse: the 4-cycle, written C_4.

That’s all the graph theory vocabulary we need to start. Everything in this project is an ordinary graph like this — the only twist is that our “dots” will turn out to be matrices.

Part 2 — Walks, paths, cycles, and “Hamiltonian”

  • A walk is a sequence of vertices, each adjacent to the next (you “walk” along edges).
  • A path is a walk that never repeats a vertex.
  • A cycle is a path that returns to its start (a closed loop), like A→B→D→C→A above.

Now the star concept:

  • A graph is Hamiltonian if it has a cycle that visits every vertex exactly once (a Hamilton cycle). Named after Hamilton, who studied this on the dodecahedron.
  • A Hamilton path is the path version: visits every vertex exactly once, but doesn’t have to return to start.

In C_4 above, A→B→D→C→A is a Hamilton cycle, so C_4 is Hamiltonian.

Why anyone cares: a Hamilton cycle is a way to list all the objects (vertices) so that consecutive ones differ by a single allowed “move.” That’s exactly a combinatorial Gray code — and that’s the broad family this whole problem lives in.

Deciding whether a general graph is Hamiltonian is famously hard (NP-complete). So proving an entire infinite family of graphs is always Hamiltonian is a real theorem, not a computation.

Part 3 — Bipartite graphs and the colour rule

A graph is bipartite if you can colour every vertex black or white so that every edge joins a black to a white (no edge inside a colour class). The square C_4 is bipartite: colour A,D black and B,C white — every edge is black–white.

Classic fact: bipartite ⇔ no odd-length cycle. (If you 2-colour along any cycle, the colours alternate, so the cycle must have even length.) Remember this — it comes back in Part 14, because for our special graphs something stronger is true.

One consequence we’ll lean on hard: in a bipartite graph, any path alternates colours. So a path with an even number of vertices starts and ends on opposite colours; odd number, same colour. This simple parity fact secretly drives the whole proof.

Part 4 — The three Hamilton properties we care about

There’s a ladder of “how freely can you thread a single path through all vertices,” weakest first:

  1. Hamiltonian — there exists one Hamilton cycle. (This is what Brualdi asked.)
  2. Hamilton-connected — for every pair of vertices u, v there’s a Hamilton path from u to v. Far stronger: you may start and end anywhere.
  3. Hamilton-laceable — the bipartite version of #2. In a bipartite graph you cannot have a Hamilton path between two same-colour vertices when the colour classes are equal-sized (parity from Part 3 forbids it). So the best possible is: a Hamilton path between every opposite-colour pair. A bipartite graph with that property is laceable.

We bundle the top of the ladder into one word:

Maximally Hamiltonian = Hamilton-laceable if the graph is bipartite, Hamilton-connected if it isn’t. I.e. the strongest path property the graph is allowed to have.

What these actually look like (small, checkable examples)

It’s worth seeing that “a Hamilton path between every pair” is real, not impossible. The trick: when your two endpoints happen to be neighbours, you may not hop directly between them (that would end the path immediately, skipping everyone) — so the path takes a scenic route. Hamilton- connectedness just says a scenic route always exists. (Every path below is real, machine-checked.)

K₄ — the complete graph on 4 vertices (every pair joined). Total freedom: pick a start, pick an end, list the other two between them in any order — every step is automatically an edge.

0→1: 0-2-3-1     0→2: 0-1-3-2     0→3: 0-1-2-3
1→2: 1-0-3-2     1→3: 1-0-2-3     2→3: 2-0-1-3

So with enough edges it’s clearly possible. But you don’t need a complete graph:

The triangular prism — sparse, yet still Hamilton-connected. Two triangles joined by three rungs; every vertex has only 3 neighbours (9 edges total):

top triangle:        0 — 1 — 2 — 0
bottom triangle:     3 — 4 — 5 — 3
rungs (a matching):  0–3,  1–4,  2–5

Watch the rerouting when the endpoints are adjacent (0 and 1 are neighbours, so the path can’t use edge 0–1):

0→1:  0-2-5-3-4-1      (skips the direct edge, snakes through all six)
0→3:  0-1-2-5-4-3
3→4:  3-5-2-0-1-4

All 15 pairs work. The lesson: you need enough alternative routes, not completeness.

Why bipartite graphs can only be “laceable.” 2-colour a bipartite graph black/white so every edge is black–white. Any path then alternates colours, so a Hamilton path through all N vertices (N/2 of each colour) must start and end on opposite colours. A Hamilton path between two same-colour vertices is therefore impossible — pure parity, not a defect of the graph. The best a bipartite graph can manage is a Hamilton path between every opposite-colour pair: that’s laceable.

K₃,₃ — complete bipartite, 3 + 3 (we meet it again later as CT₃). Top a1,a2,a3; bottom b1,b2,b3; every top joined to every bottom:

opposite-colour (a Hamilton path exists):  a1→b1:  a1-b2-a2-b3-a3-b1
same-colour (impossible, by parity):       a1→a2:  none

The cube Q₃ — 8 corners = 3-bit strings, edge = flip one bit (a Hamilton path here is literally a Gray code):

000→111:  000-001-011-010-110-100-101-111   ✔ opposite parity
000→011:  none                               ✗ same parity (both even # of 1s)

Both are laceable — every opposite-colour pair connects, no same-colour pair can.

Hold onto the distinction between “one cycle” (Hamiltonian) and “a path between every legal pair” (the strong versions). The jump between them is the whole game (Part 15 explains why we want the harder one).

Part 5 — Flip graphs: the general idea

Here’s the unifying idea behind the whole subject.

Take some family of combinatorial objects (think: all ways to triangulate a polygon, all permutations, all standard Young tableaux, all binary strings…). Define a simple, local “flip” move that turns one object into another. Now build a graph:

  • vertices = the objects,
  • edges = pairs that differ by a single flip.

That graph is a flip graph (also called a “reconfiguration graph” or, when the flips are Markov-chain moves, a “state graph”). The associahedron (triangulation flips) and the permutohedron (adjacent-transposition flips) are famous examples.

Natural questions about a flip graph: Is it connected (can you always reach any object from any other by flips)? What’s its diameter? Is it Hamiltonian — can you list every object so consecutive ones differ by one flip (a Gray code)?

Our problem is exactly a flip-graph Hamiltonicity question, for one specific, important family of objects and one specific flip. Let’s meet them.

Part 6 — Matrices with fixed margins (what “margins” means)

Our objects are matrices of 0s and 1s with prescribed row sums and column sums. First the word margins, because it’s central and it’s just jargon for something simple.

Take a matrix, add up each row and write those totals down the right edge; add up each column and write those totals along the bottom edge. Those edge totals are the margins (statisticians call them marginal totals — they literally sit in the margins of the table):

         c1 c2 c3  |  row sums  = R
   r1     1  0  1  |     2
   r2     0  1  1  |     2
   r3     1  1  0  |     2
         ---------
  col      2  2  2
  sums =  S

The “2 2 2” down the right is the row-sum vector R = (2,2,2); the “2 2 2” along the bottom is the column-sum vector S = (2,2,2). Together, R and S are the margins of the matrix.

Now fix the margins and gather every 0/1 matrix that has them:

  • Fix a row-sum vector R = (r_1, ..., r_m) and a column-sum vector S = (s_1, ..., s_n).
  • A(R,S) = all m×n matrices of 0s and 1s whose row sums are exactly R and column sums exactly S. (The matrix above is one member of A((2,2,2),(2,2,2)); there are others with the same margins — e.g. shift the 1s around while keeping every row and column summing to 2.)

Where the idea comes from: a grid of counts with fixed row and column totals is a contingency table (statistics — e.g. a survey cross-tabulated by two attributes, with the attribute totals held fixed). The very same objects are transportation plans (ship from m sources with supplies R to n destinations with demands S) and bipartite degree sequences (Part 9). Brualdi’s question, in plain words: can you walk through all tables with given margins, one small move at a time?

Worked example we’ll reuse. Let m = n = 3, and R = S = (1,1,1) — every row and every column sums to 1. A 0/1 matrix with exactly one 1 per row and per column is a permutation matrix. There are 3! = 6 of them. Here are two:

1 0 0          0 1 0
0 1 0          1 0 0
0 0 1          0 0 1

The left is the identity. The right swaps rows’ 1s in columns 1 and 2. So A((1,1,1),(1,1,1)) has 6 elements — the 6 permutation matrices of size 3.

For bigger or uneven margins the class A(R,S) is larger and far less symmetric — but it’s always the same idea: fixed margins, 0/1 entries.

Part 7 — The 2×2 interchange move

Now the flip. A 2×2 interchange (a.k.a. swap or 2-switch) picks two rows and two columns where the 2×2 corner submatrix is one of these two patterns and flips it to the other:

1 0      <--->      0 1
0 1                 1 0

Everything outside those four cells stays fixed. Notice: each row of the little block keeps its sum (1 stays 1), and each column keeps its sum too. So an interchange preserves all the margins — it turns one matrix in A(R,S) into another matrix in the same A(R,S).

Concrete example on a 3×3 matrix. Take the identity and interchange using rows {1,2}, columns {1,2}:

1 0 0          0 1 0
0 1 0    -->   1 0 0
0 0 1          0 0 1

The top-left 2×2 block went from [[1,0],[0,1]] to [[0,1],[1,0]]; row 3 / column 3 untouched. Both matrices have margins (1,1,1)/(1,1,1). One flip, one new matrix.

Foundational fact (Ryser, 1957): any two matrices in A(R,S) can be turned into each other by a sequence of these interchanges. In flip-graph language: the flip graph is connected. Good — connectivity is the floor; Brualdi conjectured the ceiling (Hamiltonicity).

Part 8 — The interchange graph (our flip graph), by hand

Now assemble the flip graph for our objects + move:

Interchange graph G(R,S): vertices = the matrices in A(R,S); an edge joins two matrices that differ by a single 2×2 interchange.

It’s an ordinary simple graph (Part 1). The fact that the dots are matrices is just labelling.

Let’s build a tiny one fully. Take m=n=2, R=S=(1,1). Then A has just two matrices:

1 0          0 1
0 1          1 0

They differ by one interchange (the whole matrix is the 2×2 block). So G is two vertices joined by one edge — the single-edge graph K_2.

Now the first interesting one: R=S=(1,1,1) (3×3 permutation matrices). Six vertices (Part 6). Two permutation matrices are adjacent exactly when one is obtained from the other by swapping two columns’ worth of 1s — i.e. when the two permutations differ by one transposition (one swap).

  • The identity 123 is adjacent to 213, 132, 321 (swap any one pair). Degree 3.
  • Every permutation has exactly 3 single-swaps, so the graph is 3-regular on 6 vertices.
  • Colour each permutation by its parity (even/odd number of swaps to sort it). A single swap flips parity, so every edge joins an even to an odd permutation — the graph is bipartite, 3 of each colour.
  • A 3-regular bipartite graph on 3+3 vertices, with every even adjacent to every odd… that’s exactly K_{3,3}, the complete bipartite graph.

So G((1,1,1),(1,1,1)) = K_{3,3}. Keep this example in your pocket — it’s the smallest rich case, and K_{3,3} will literally reappear as a building block (it’s “CT_3” in Part 12).

Sanity check you can do on the phone: K_{3,3} has a Hamilton cycle (e1→o1→e2→o2→e3→o3→e1, zig-zag across the colours). So this interchange graph is Hamiltonian — Brualdi’s conjecture holds for these margins. The conjecture says: no matter the margins, you can always do this.

Part 9 — The same object in three other languages (optional)

You can skip this part and lose nothing essential — it’s just three other ways people describe the exact same graph. But one of them may make it click, and the third is the one graph theorists usually find most natural.

(1) A Markov-chain state graph. To sample random contingency tables with fixed margins, statisticians start from one table and repeatedly apply random 2×2 interchanges — a random walk on the tables. G(R,S) is the “state graph” of that walk: states = tables, moves = interchanges.

(2) The 1-skeleton of a transportation polytope. Quick refresher: a polytope is the higher-dimensional analogue of a polygon (2D) or polyhedron (3D) — the solution set of some linear inequalities, with flat faces, straight edges, and corner vertices. Now relax our matrices: instead of 0/1 entries, allow any nonnegative real entries that still produce the same margins R and S. That set of real matrices forms a convex polytope — the transportation polytope (from the “transportation problem”: ship goods from sources with supplies R to destinations with demands S; the matrix says how much travels on each route). Two facts tie it back to us:

  • its corner vertices are exactly the 0/1 matrices (the integer extreme points), and
  • two corners are joined by an edge of the polytope precisely when they differ by one 2×2 interchange.

The 1-skeleton of a polytope means: keep only its vertices and edges, throw away the 2-dimensional and higher faces — i.e. the plain graph of corners and edges. (Example: the 1-skeleton of an ordinary 3D cube is the 8-vertex “cube graph.”) So G(R,S) is the 1-skeleton of the transportation polytope with margins R, S. That’s the only sense in which anything “higher-dimensional” is involved: the polytope has higher faces, but the interchange graph itself is just corners and edges — an ordinary graph.

(3) Bipartite graphs under 2-switches (the friendly one for graph theorists). A bipartite graph splits its vertices into two sides, with edges only between the sides. Record one by a biadjacency matrix: rows = left-side vertices, columns = right-side vertices, and a 1 in cell (i,j) means “there is an edge from left-i to right-j” (0 = no edge). Then:

  • the row sums and column sums are exactly the degrees (edge-counts) of the vertices on each side, so fixed margins = a fixed degree sequence on each side;
  • a 2×2 interchange is exactly a 2-switch: remove edges i–b and j–a, add edges i–a and j–b — the classic move that rewires a graph while keeping every vertex’s degree the same.

So G(R,S) is the flip graph of all bipartite graphs with a prescribed degree sequence, under 2-switches — what graph theorists call a realization graph of the degree sequence. Graph theorists work on graph Hamiltonicity, so this is the dictionary that let that intuition transfer straight in.

Picture for #3 — a 2-switch on a little bipartite graph (left side 1,2; right side a,b):

 1 — a            1   a
        becomes      X
 2 — b            2   b      (edges 1a,2b  ->  1b,2a)

Same degrees on every vertex, different graph. That’s one edge of G(R,S).

Part 10 — Brualdi’s conjecture and its history

Brualdi’s conjecture (1980). For all margins R, S, the interchange graph G(R,S) is Hamiltonian.

In words: the matrices with any fixed margins can always be listed so that consecutive ones differ by a single 2×2 interchange — a Gray code for contingency tables.

What was known before us:

  • Li & Zhang (1994): true when one margin is all-1s. Their tools (induction + Cartesian products
    • Hamiltonicity of a matroid basis graph) are, strikingly, the same family of tools we use — they just handled that special case.
  • Arikati & Peled (1999): true for “majorization gap 1” degree sequences — another special case, via the realization-graph view of Part 9.
  • Otherwise open for 45 years. It’s Problem P60 in Mütze’s survey Combinatorial Gray Codes (2024), the standard catalogue of open problems in this area.

So the target is a recognized, named, long-standing open problem with only fragments solved. That’s why (if our proof holds) the journal ambitions are high.

Part 11 — Cartesian products of graphs

We need one standard graph construction, because our hard cases are built from smaller graphs.

The Cartesian product G □ H of two graphs:

  • vertices = ordered pairs (g, h) with g in G and h in H;
  • (g,h) is adjacent to (g',h') exactly when one coordinate matches and the other is an edge — i.e. (g=g’ and h~h’) or (h=h’ and g~g’).

Think of a grid: move one coordinate at a time. Examples:

  • K_2 □ K_2 = the square C_4 (a 2×2 grid).
  • K_2 □ K_2 □ K_2 = the cube Q_3. In general K_2 to the box-power k = the k-dimensional hypercube Q_k (binary strings, flip one bit — itself a famous flip graph / Gray code!).
  • (path) □ (path) = a rectangular grid graph.

Key intuition for later: a Cartesian product is “layers of G, indexed by H, stitched together.” To thread a single path through the whole product you traverse layers and hop between them — and that hopping is where the strong path properties (Part 4) matter, because a plain Hamilton cycle doesn’t survive this stitching but Hamilton-connectedness/laceability do.

Part 12 — Complete transposition graphs

A special family of interchange graphs gets its own name.

Complete transposition graph CT_a: take the a! permutations of a symbols; join two if they differ by one transposition (one swap of two symbols). Equivalently, it’s the Cayley graph of the symmetric group S_a with all transpositions as generators. Equivalently again, it’s G(R,S) for the all-1s a×a margins.

We’ve already met them:

  • CT_2 = K_2 (one edge).
  • CT_3 = K_{3,3} (Part 8).
  • CT_4 has 4! = 24 vertices, and so on.

Each CT_a is bipartite (parity again), vertex-transitive, and very symmetric. These are the “atoms.” The structure theorem (next part) says the bipartite interchange graphs are exactly Cartesian products of these atoms, e.g. CT_3 □ CT_4. Those products are the single hard family the whole proof funnels into.

Part 13 — The structure theory: lines, fibres, quotients

To run an induction on these graphs, we need to know how they’re shaped. Three ideas; learn the words, not the fine print.

  • Line = a row or a column of the matrix.
  • Invariant cell = a cell that has the same value in every matrix of A(R,S). Such a cell never takes part in any interchange (it can’t — it’s frozen). Delete all invariant cells and you get the invariant-free core; its interchange graph is the same graph, just described more cleanly. We always reduce to the invariant-free case. (“Active” cell = not invariant.)
  • Fibre and quotient. Freeze one line L (say, fix the first row to a specific 0/1 pattern).
    • The matrices that share a given everything-except-L form a fibre — and a fibre is itself a smaller interchange graph.
    • The collection of all legal patterns the line L can take, joined by single interchanges inside L, is the quotient Q_L.
    • So G(R,S) looks like: copies of fibres (smaller versions of the same kind of object), laid out over the quotient Q_L, glued along edges. This “split along a line into fibres + a quotient” is the engine of the induction.

One deep bonus fact you only need to recognize: the legal patterns of a line are the bases of a matroid, so Q_L is a matroid basis-exchange graph. That matters because there’s a powerful theorem (Naddef–Pulleyblank) that such graphs are always either hypercubes or Hamilton-connected — which the proof cashes in at Part 17.

Part 14 — The bipartite = triangle-free surprise

This is a natural first point of confusion, so it’s worth getting crisp.

A triangle is a 3-cycle (three mutually adjacent vertices). “Triangle-free” = no triangles.

For graphs in general: bipartite ⇔ no odd cycle (Part 3). Being triangle-free is weaker — you can be triangle-free but still have a 5-cycle, so still not bipartite. That is exactly right. (The 5-cycle C_5 is the standard example: no triangle, not bipartite.)

But interchange graphs are special. Brualdi proved (Thm 6.3.4 in his book Combinatorial Matrix Classes): an interchange graph that contains any odd cycle already contains a triangle. So for this class — and only because of its rigid structure — triangle-free does imply bipartite, and the two notions coincide.

We double-checked by computer: across all 6,017 interchange graphs with margins up to 6×6, every non-bipartite one has a triangle and every bipartite one is triangle-free — 0 exceptions.

And Brualdi’s theorem says more: for an invariant-free class, bipartite ⇔ triangle-free ⇔ the class is a Cartesian product of complete transposition graphs (and complements). That last clause is the structural gold we use next.

Part 15 — The strategy: strengthen, then reduce to one lemma

Two moves make the whole proof.

Move 1 — strengthen the goal (counter-intuitive but essential). We prove not “Hamiltonian” but “maximally Hamiltonian” (Part 4). Why prove something harder? Because the proof builds big graphs from small ones (Cartesian products in Part 11, fibres+quotients in Part 13), and:

  • a Hamilton cycle does not survive that gluing, but
  • Hamilton-connectedness and laceability do.

So the stronger statement is the one that can be passed up an induction. (This “strengthen the inductive hypothesis” trick is standard.)

Move 2 — reduce everything to one lemma. Induct on the number of vertices. The (invariant-free) graph is always one of three kinds — a trichotomy:

  1. a nontrivial Cartesian product A □ B → handle by gluing (Parts 11, 16);
  2. indecomposable but not a tiny base case → the pivot construction (Part 17);
  3. a small base case → check directly / cite.

Cases 1 and 2 use the smaller pieces (induction hypothesis) plus one hard input about the bipartite atoms:

Block Lemma. Every balanced bipartite interchange graph (= every Cartesian product of CT_a’s) has a strong path-covering property called paired 2-disjoint-path-coverability.

Everything funnels into this lemma.

What that property means (call it “paired-2-DPC”): pick two pairs of endpoints, (a1,b1) and (a2,b2). You can always find two vertex-disjoint paths — one from a1 to b1, one from a2 to b2 — that together cover every vertex. Requiring this for every choice of the two pairs is paired-2-DPC.

It’s strictly stronger than laceable (laceable = the one-pair version), and — crucially — it’s exactly the property that survives Cartesian products. That’s why it, and not bare Hamiltonicity, is the right inductive currency.

Part 16 — The engine: Coleman 2024 + the double induction

How do we prove the Block Lemma (products of CT_a are paired-2-DPC)? The key input is new, which is why this proof wasn’t possible until recently:

Coleman, Fischberg, Gong, Harrington, Wong (2024). A broad class of bipartite graphs they call “transposition-like” have paired (n−1)-disjoint-path covers (the n−1 version of the property above). Each CT_a is one of their examples.

The unlock: in their theorem, the bottom-level building blocks (“leaves”) may be any even-sized graph that is Hamilton-laceable. So we can take an entire product of CT_a’s and feed it in as a single leaf. Then a double induction finishes:

  • Outer induction on the number of factors in the product. It needs just one fact about the smaller product B: that B is Hamilton-laceable (which follows because paired-2-DPC ⇒ laceable).
  • Inner induction is Coleman’s own induction, with B fixed as the leaf.

The two inductions move on different parameters and don’t tangle (no circular reasoning). This part is also machine-checked in the Lean proof assistant (the recursion is verified, no gaps).

In short: “Blocks are products of CT_a; Coleman 2024 gives the paired cover whenever the leaf is any even laceable graph, so a whole CT-product folds in as a leaf, and a clean double induction closes it.”

Part 17 — The pivot construction (our deepest part)

This handles trichotomy case 2: the graph is indecomposable (not a product) and not a tiny base case. It’s the most original part — and the part most needing independent scrutiny, because it’s the one piece the Lean formalization does not verify (it’s assumed there).

Goal: a Hamilton path between two chosen vertices a and b. The idea, in four beats:

  1. Pick a pivot line L that “separates” a and b — a line where the two matrices differ. (A short argument shows such a line exists unless the graph is tiny.)
  2. Look at the quotient Q_L (Part 13) — the small graph of legal patterns of that line. A matroid argument (using “shifted matroids”) shows Q_L is non-bipartite here.
  3. Q_L is Hamilton-connected — because it’s a matroid basis-exchange graph, and Naddef– Pulleyblank says those are hypercubes or Hamilton-connected (and we’ve just excluded the hypercube case via non-bipartiteness).
  4. Route and lift. Take a Hamilton path through Q_L from a’s pattern to b’s pattern; each step corresponds to a whole fibre (a smaller interchange graph), which by the induction hypothesis can itself be traversed by a Hamilton path. Stitch the fibres together along the quotient route → one Hamilton path through the entire graph.

The delicate part is the bookkeeping at the splices — making the colours/parities line up at every junction so the fibre-paths actually join into a single path. That’s where an error would hide, and it’s the first thing to scrutinise.

In short: “Pivot on a separating line; its quotient is a Hamilton-connected matroid base graph; route through it and lift fibre-by-fibre using the induction hypothesis, matching parities at the splices.”

Part 18 — The bonus corollary: two communities

Recall Part 9: a 0/1 matrix is a bipartite graph, fixed margins = fixed degrees, an interchange = a 2-switch. So G(R,S) is literally the realization graph (2-switch flip graph) of a bipartite degree sequence.

Therefore our theorem also settles Hamiltonicity of those realization graphs — a question recorded as open in the degree-sequence community (Barrus, 2016), in the bipartite / split-graph case (a split graph is one whose vertices divide into a clique plus an independent set).

An honest caveat: it’s the same theorem in two languages, not a stronger one. The fully general version — realization graphs of arbitrary degree sequences, not just bipartite — stays open. But it means the result speaks to two research communities at once, which helps its appeal.

Part 19 — How we know it’s (probably) right — and the honest gaps

Evidence, strongest first:

  • Direct computer check of the theorem itself. We built 144,389 of these graphs (all margins up to 7×7 reachable in a 2-hour run) and verified maximal Hamiltonicity by SAT-solving every required Hamilton-path instance — 2,555,559 of them — with zero counterexamples. This tests the conclusion directly, so it would catch a mistake anywhere in the argument. It found none.
  • AI adversarial review. Many rounds of independent “try to break it” passes (Claude + Codex + Gemini), which did catch and fix real errors along the way. A strong filter, not a proof.
  • Partial Lean formalization. The logical assembly and the Block-Lemma double induction (Part 16) are machine-checked, with no gaps.

The honest gaps:

  • The §§3–6 reduction, especially the pivot construction (Part 17) and the product-lifting parity bookkeeping (Part 16), is not machine-proved. In Lean it’s entered as an assumption. So it rests on the AI review + the computations — this is exactly the part that still needs independent human verification.
  • “Checked on 144k graphs” is corroboration on a finite range, not a proof. Important not to oversell that.

One-sentence status: strong, multi-layered evidence; a candidate proof; awaiting human-expert verification of the reduction.

Part 20 — Glossary (one-liners)

  • Graph / vertex / edge / adjacent / degree: dots, lines, “joined,” number of neighbours.
  • Walk / path / cycle: edge-sequence / no repeats / closed loop.
  • Hamiltonian: has a cycle through all vertices. Hamilton path: path through all vertices.
  • Bipartite: 2-colourable so every edge is black–white; equivalently, no odd cycle.
  • Hamilton-connected: a Hamilton path between every pair. Laceable: the bipartite analogue (every opposite-colour pair). Maximally Hamiltonian: laceable if bipartite, else Hamilton-connected.
  • Flip graph: vertices = combinatorial objects, edges = single local “flip” moves.
  • Margins (marginal totals): the fixed row-sum vector R and column-sum vector S — the totals written down the right edge and along the bottom edge of the matrix.
  • Contingency table: a grid of counts cross-classified by two attributes; “fixed margins” means the row totals and column totals are held fixed.
  • Polytope / transportation polytope: a polytope is the higher-D analogue of a polygon (corner vertices, straight edges, flat faces). The transportation polytope for margins R,S is the set of nonnegative real matrices with those margins; its corner vertices are exactly the 0/1 matrices.
  • 1-skeleton: a polytope’s vertices-and-edges only (higher faces discarded); G(R,S) is the 1-skeleton of the transportation polytope with margins R, S.
  • Biadjacency matrix: the 0/1 matrix encoding a bipartite graph — rows = left vertices, columns = right vertices, a 1 means an edge.
  • A(R,S): all 0/1 matrices with margins R, S. 2×2 interchange / 2-switch: flip a 2×2 identity ↔︎ anti-identity; preserves margins.
  • Interchange graph G(R,S): the flip graph — vertices A(R,S), edges single interchanges.
  • Cartesian product G □ H: pairs of vertices; move one coordinate at a time.
  • CT_a (complete transposition graph): permutations of a items joined by single swaps; the all-1s a×a interchange graph; CT_2=K_2, CT_3=K_{3,3}.
  • Balanced bipartite block: a Cartesian product of CT_a’s; the hard core of the problem.
  • Paired-2-DPC: for any two endpoint-pairs, two disjoint paths covering all vertices; stronger than laceable; survives products.
  • Line / fibre / quotient: a row or column / the smaller graph from fixing everything else / the graph of a line’s legal patterns (a matroid base graph).
  • Pivot construction: prove Hamilton paths in indecomposable classes by routing through a separating line’s quotient and lifting fibre-by-fibre.
